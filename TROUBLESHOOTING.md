# Устранение Internal Server Error

## Шаг 1: Очистка кэша и перезапуск

Выполните в терминале **в папке проекта**:

```bash
cd "/Users/zakharkorchagin/My project/relationship-app"

# Остановите сервер (Ctrl+C), затем:

# Удалить кэш сборки Next.js
rm -rf .next

# Перезапустить
npm run dev
```

Откройте http://localhost:3000

- **Если видите «Если вы видите это — сервер работает»** — сервер в порядке, проблема была в коде приложения. Напишите об этом, и мы вернём полный интерфейс по шагам.
- **Если снова Internal Server Error** — переходите к шагу 2.

## Шаг 2: Посмотреть ошибку в терминале

При открытии http://localhost:3000 в **терминале**, где запущен `npm run dev`, часто появляется текст ошибки (красный или стек вызовов).

Скопируйте **полный текст ошибки** из терминала и пришлите его — по нему можно точно найти причину.

## Шаг 3: Проверка порта и других приложений

Если раньше вы меняли что-то, связанное с localhost:3000:

- **Другой процесс на порту 3000** — закройте другие приложения/серверы и снова запустите `npm run dev` или запустите на другом порту: `npm run dev -- -p 3001` и откройте http://localhost:3001
- **Прокси / VPN / антивирус** — попробуйте отключить на время или открыть сайт в режиме инкогнито.
- **Расширения браузера** — откройте http://localhost:3000 в режиме инкогнито или в другом браузере.

## Шаг 4: Полная переустановка зависимостей

```bash
cd "/Users/zakharkorchagin/My project/relationship-app"
rm -rf .next node_modules package-lock.json
npm install
npm run dev
```

После этого снова откройте http://localhost:3000 и, если ошибка остаётся, пришлите **точный текст из терминала** при открытии страницы.
